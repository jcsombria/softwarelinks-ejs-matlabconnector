Ejs-Labview model element

A Labview model element encapsulates the connection, via JIL server, with LabVIEW software to control the execution of a VI.

Usage

To add a LabVIEW model element to your model, drag the icon element to the list of your model elements and set the name of the variable for referring to the element. The connection can be configured in the preferences window, the address of the server, the port to establish the connection and the path to the LabVIEW VI. The links between the EJS variables and the controls/indicators of LabVIEW also can be done in this window.

The main methods of the LabVIEW class are:

  • connect() Creates a new connection with JIL Server.
  • openVI() Opens the VI specified in path.
  • runVI(boolean wait) Runs the opened VI.
  • step() Sends the new values of the controls and get the values of the indicators.
  • stopVI() Stops the running VI.
  • closeVI() Closes the VI.
  • disconnect() Disconnects from JIL Server.
A call to the method setValues() A call to the element method Boolean isConnected() returns true if the connection with JIL is opened and correctly working, and false otherwise. A call to the element method Boolean isRunning() returns true if the LabVIEW VI is running and false if it is stopped.

Example of use

The following code is usually included into an initialization page, and it must be executed to set up the connection (we use labview as instance name of the element) 

labview.connect();
labview.openVI();
labview.runVI(false);

The following code is usually included into an evolution page, to exchange data periodically with the server. 

labview.step();

The following code must be executed to finalize the execution properly. 

labview.stopVI();
labview.closeVI();
labview.disconnect();
